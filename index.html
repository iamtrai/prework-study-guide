HTML

  • HTML stands for Hypertext Markup Language
  • HTML contains all of the base information for a website to use without much stylising or functionality
  • Tag refers only to what appears inside the angle brackets
  • Element refers to the entire code from opening tag to closing tag
  • The head element contains information about the webpage.
  • The meta element contains information about the page that is used by the browser
  • The body element represents the visible content shown to the user.
  • The h1, h2, h3, h4, h5, and h6 elements represent different sizes of headers of text, and should be used in that order with h1 generally being used no more than once.

CSS

  • CSS stands for Cascading Style Sheets
  • CSS defines how webpage content is presented to users with regard to colors, aesthetics, visual layout, and organization.
  • There are three ways to style a webpage with CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • When a developer wants to style an element directly in the HTML file, they use inline CSS.
  • Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • Useful for quick and permanent changes but less flexible than external and internal style sheets - each inline style must be edited individually whenever you make a design change
  • It is possible to embed an entire stylesheet directly in HTML using the style element in the head element.
  • Use of external CSS style sheets are most common and preferred because it allows develoeprs to keep all their CSS rules in a separate file which makes design changes easier.
  • When using an external style sheet, we link the file using a link element placed in the HTML's head element
  • A selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property. For example, the declaration color: blue, includes the CSS property color and the value for that property, blue.
  • The margin property indicates how much space we want around the outside of an element.
  • The padding property is often confused with margin. The padding property adds space around the content inside of an element. See CSS box model
  • When you want to apply a rule to all elements visible on the page, use a * symbol for the selector.
  • Rather than repeat a rule, two or more elements that will share the same styling can be grouped together by combining their selectors into a single CSS rule with a comma.
  • Do Not Repeat Yourself or DRY coding encourages developers to reuse or share pieces of code in order to reduce the number of lines of code that need to be written or maintined
  • You can also designate rules to apply to specific elements using class selectors. Add a preiod before your slector to designate it as a class selector. Any HTML element assigned to the class will gain those properties.

JavaScript

  • JavaScript is a high-level dynamic programming language that allows useres to interact with websites.
  • JS powers websites with effects, interactivity, data communication, and much more.
  • APIs are Application Programming Interfaces. Provide functionality such as dynamically creating HTML and setting CSS styles; collecting and manipulating a video stream from a user's webcam, or generating 3d graphics and audio samples.
  • Third-party APIs that allow developers to incorporate functinality in sites from other content providers, such as Twitter or Facebook.
  • Third-party frameworks and libraries that you can apply to HTML to accelerate the work of building sites and applications.
  • JavaScript is a dynamic language with dynamic types. Variables in JavaScript are not directly associated with any particular value type, and any variable can be assigned (and reassigned) values of all types.
  • JavaScript is also a weakly typed language, meaning it allows implicit type conversion when an operation involves mismatched types, instead of throwing typing errors.
  • In JavaScript, to declare a variable, we use the var keyword. After the var keyword, we give the variable a name. The name must be unique because it is how we will reference the information stored in the variable in our code.
  • Variables are containers that store values. You start by declaring a variable with the let keyword, followed by the name you give to the variable. i.e. let myVariable;
  • A semicolon at the end of a line indicates where a statement ends. It is only required when you need to separate statements on a single line, though some believe it is good practice to have semicolons at the end of each statement.
  • You can name a variable nearly anything, but there are some restrictions.
  • JavaScript is case sensitive. myVariable is not the same as myvariable. If you have problems with your code, check the case.
  • After declaring a variable, you can give it a value: myVariable = "Bob"; You can also do both these operations on the same line: let myVariable = "Bob";
  • You retrieve the value by calling the variable name: myVariable; After assigning a value to the variable, you can change it later in the code: let myVariable = "bob"; myVariable = "Steve";
  • Control Flow is the order in which a computer executes code in a script. In JavaScript, code runs from the first statement at the top of the page, down to the last statement at the bottom of the page.
